Appendix: Signal Configuration Fields

Each entry in the signals array of the config file describes one channel. The following explains what each field means, illustrated with one analog and one digital example.

Analog Example — DPT (Discharge Pressure Transmitter)

{
    "name": "DPT",
    "sig_type": "ao",
    "boardSlotPosition": 0,
    "engineeringUnits": "PSI",
    "engineeringUnitsLowAmount": 0.0,
    "engineeringUnitsHighAmount": 200.0,
    "showOnGUI": true,
    "offset_calib_constant": 0,
    "slope_calib_constant": 1
}
  • name — The label used to identify the signal throughout the software and on the GUI. This is the key the application uses internally to look up the channel.
  • sig_type — The signal type and direction. "ao" means analog output (the simulator sends a 4–20 mA signal to the control panel). The four valid values are "ao", "ai", "do", and "di".
  • boardSlotPosition — The physical slot number of the module on the carrier board. The software uses this to compute which SPI bus, card, and slot to address when sending or reading the signal. Analog and digital outputs share one SPI bus; inputs share another.
  • engineeringUnits — The unit label displayed alongside the signal value on the GUI (e.g. PSI, Amps, %). This is display-only and does not affect calculations.
  • engineeringUnitsLowAmount — The real-world value that corresponds to 4 mA (the bottom of the 4–20 mA range). For DPT this is 0.0 PSI, meaning a 4 mA output represents 0 PSI.
  • engineeringUnitsHighAmount — The real-world value that corresponds to 20 mA (the top of the range). For DPT this is 200.0 PSI, meaning a 20 mA output represents 200 PSI. All intermediate values are linearly interpolated between these two bounds. Note that low and high can be swapped to represent an inverted signal (e.g. a sensor that reads high at low pressure).
  • showOnGUI — Controls whether this signal appears as an interactive widget on the main display. Spare or unused channels are set to false to keep the interface uncluttered; they can still be addressed programmatically.
  • offset_calib_constant — (Analog inputs only) The additive offset b in the calibration formula y = mx + b, applied to the raw mA reading before converting to engineering units. Used to correct for a sensor that reads consistently high or low. A value of 0 applies no offset correction.
  • slope_calib_constant — (Analog inputs only) The multiplicative slope m in the same calibration formula. Used to correct for a sensor whose output scale is slightly off. A value of 1 applies no slope correction. Together with offset_calib_constant, these two constants define a linear correction that is applied before the mA-to-engineering-units conversion.

Digital Example — MTR1 (Motor 1)

{
    "name": "MTR1",
    "sig_type": "do",
    "boardSlotPosition": 32,
    "engineeringUnits": null,
    "engineeringUnitsLowAmount": 0,
    "engineeringUnitsHighAmount": 1,
    "showOnGUI": true
}
  • name — Same as analog: the identifier used by the software and displayed on the GUI.
  • sig_type — "do" means digital output (the simulator drives a relay on or off). The signal carries only two states: on or off.
  • boardSlotPosition — Same as analog: the physical slot number used to route the command to the correct hardware module. Digital output slots begin at 32 per the board layout.
  • engineeringUnits — null for all digital signals. There is no continuous physical unit; the signal is simply on or off.
  • engineeringUnitsLowAmount — Always 0 for digital signals, representing the off (open) state.
  • engineeringUnitsHighAmount — Always 1 for digital signals, representing the on (closed) state.
  • showOnGUI — Same as analog: whether the signal appears as a control on the main display.

Digital signals do not use calibration constants. The value sent or received is always either 0 or 1.
